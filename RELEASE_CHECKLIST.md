# HandyToFile — чеклист релиза

## 1. Окружение для сборки

- Проверить наличие Rust тулчейна.
- Проверить наличие Node.js и npm.
- Проверить установку Visual Studio C++ build tools.
- Проверить, что `LIBCLANG_PATH` указывает на папку `bin` LLVM.
- Проверить, что `VULKAN_SDK` указывает на установленный Vulkan SDK.

## 2. Установка зависимостей

```powershell
cd C:\projects\HandyToFile
& 'C:\Program Files\nodejs\npm.cmd' install --ignore-scripts
```

## 3. Обязательные проверки перед сборкой

```powershell
cd C:\projects\HandyToFile
& 'C:\Program Files\nodejs\npm.cmd' run build

cd C:\projects\HandyToFile\src-tauri
$env:LIBCLANG_PATH = 'C:\Program Files\LLVM\bin'
$env:VULKAN_SDK = 'C:\VulkanSDK\1.4.341.1'
cargo check
cargo test
```

## 4. Сборка релизного артефакта

```powershell
cd C:\projects\HandyToFile
$env:LIBCLANG_PATH = 'C:\Program Files\LLVM\bin'
$env:VULKAN_SDK = 'C:\VulkanSDK\1.4.341.1'
& 'C:\Program Files\nodejs\npm.cmd' run tauri build
```

## 5. Ручное дымовое тестирование

- Запустить собранное приложение.
- Убедиться, что основной шорткат вставки работает.
- Убедиться, что запись с микрофона запускается и останавливается корректно.
- Убедиться, что распознанный текст вставляется в активное поле ввода.
- Убедиться, что захват заметок добавляет текст в настроенный Markdown-файл.
- Убедиться, что захват задач добавляет строку вида `- [ ] YYYY-MM-DD HH:mm — текст`.
- Убедиться, что захват покупок добавляет строку вида `- [ ] YYYY-MM-DD HH:mm — текст`.
- Убедиться, что захват идей добавляет заголовок + тело + пустую строку.
- Убедиться, что пустой или состоящий только из пробелов результат распознавания не записывается в файл.
- Убедиться, что отсутствующие директории и файлы создаются автоматически.
- Убедиться, что после записи в файл показывается уведомление об успехе.
- Убедиться, что при ошибке записи в файл показывается уведомление об ошибке.
- Убедиться, что пути к файлам захвата можно изменить в настройках.

## 6. Проверка дефолтных пользовательских значений

- Убедиться, что дефолтное действие вставки включено.
- Убедиться, что дефолтные действия захвата в Markdown включены.
- Убедиться, что дефолтные пути разумны для целевой аудитории или чётко задокументированы.
- Убедиться, что `recognition_profiles` и `capture_actions` предзаполнены в конфиге.

## 7. Проверка релизного пакета

- Убедиться, что пакет содержит собранный артефакт десктопного приложения.
- Убедиться, что примечания к релизу упоминают новый рабочий процесс захвата.
- Убедиться, что ссылки в README указывают на актуальную документацию форка.
- Убедиться, что инструкции по установке для конечных пользователей включены.

## 8. Валидация установки у конечного пользователя

На чистой машине Windows:

- Установить упакованное приложение.
- Убедиться, что WebView2 присутствует, или установить его.
- Успешно запустить приложение.
- Предоставить разрешение на микрофон.
- При необходимости предоставить разрешение на доступность или управление вводом.
- Скачать или выбрать локальную модель распознавания речи.
- Выполнить то же дымовое тестирование для вставки и действий захвата в Markdown.
